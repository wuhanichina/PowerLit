# PowerLit 预期功能

本文档用于维护 `PowerLit` 的预期实现功能，区分“当前已实现”“近期目标”“中期目标”和“暂不优先”。`README.md` 侧重使用说明，本文档侧重产品能力边界与演进方向。

## 1. 项目定位

`PowerLit` 是一个面向电力系统研究的本地优先文献检索与知识整理工具。

目标不是做一个通用学术搜索引擎，而是围绕电力系统研究工作流，逐步打通下面这条链路：

`论文检索 -> 元数据规范化 -> 本地索引 -> PDF绑定 -> PDF解析 -> 结构化抽取 -> SOTA对比 -> Markdown知识库 -> 研究脉络可视化`

更长期的产品目标是：基于真实文献构建一个用于抑制 AI 幻觉的研究基础设施，而不是只做“PDF 转 Markdown”的整理工具。

这意味着 `PowerLit` 后续所有 AI 能力都应尽量建立在“已入库、可追溯、可核查”的本地文献之上，并逐步支持：

- 让 AI 可以直接调用文献库中的结构化内容
- 让结论、观点、对比结果可以回查到原文证据
- 让多篇论文可以做稳定的横向比较
- 让文献库直接服务于选题、related work 与综述写作

## 2. 研究范围

重点覆盖：

- 电力系统稳态分析与运行优化
- 暂态稳定、小扰动稳定、电压稳定、频率稳定
- 电力市场、出清、报价、辅助服务、双层博弈
- 综合能源系统、多能协同优化
- 新能源并网、储能、配电网、微电网、韧性恢复

低优先级或暂不重点覆盖：

- 高电压与绝缘
- 器件级电力电子
- 继电保护硬件实现
- 材料、装备、实验平台类论文

## 3. 当前已实现

### 3.1 文献检索与索引

- 支持 `Crossref` 与 `OpenAlex` 直接检索
- 预留 `IEEE Xplore` 与 `Elsevier Scopus` 接口
- 支持按主题预设检索
- 支持批量检索
- 支持去重与元数据合并
- 支持输出 `JSON`、`CSV`、`Markdown`
- 支持写入本地 `SQLite` 索引库

### 3.2 PDF工作流

- 支持生成全文下载队列
- 支持将手动下载的 PDF 绑定回 DOI
- 支持在 Web 页面上传并绑定 PDF

### 3.3 Web界面

- 仪表盘
- 文献库
- 下载队列
- API 文档

## 4. 近期目标

### 4.1 全文获取架构

在进入解析前，全文获取链路应先升级为正式的四级架构：

- `Level 1`: 结构化开放接口
- `Level 2`: 开放版本定位
- `Level 3`: 落地页规则提取
- `Level 4`: 浏览器自动化回退

其中浏览器自动化的定位必须是：

- 合法访问场景下的末级回退获取器
- 用于解决“最后一公里获取失败”
- 不作为主下载器，也不用于绕过授权边界

推荐整体流水线：

`OpenAlex -> 筛选与去重 -> OpenAlex 全文对象 / 链接 -> DOI -> Unpaywall -> publisher page rule-based downloader -> browser-agent fallback -> PDF / TEI -> GROBID / TEI parser -> Markdown -> LLM结构化分析`

建议实现顺序：

- `step1_search_openalex`
- `step2_resolve_fulltext`
- `step3_download_rule_based`
- `step4_download_browser_fallback`
- `step5_parse_grobid`
- `step6_build_markdown_card`

详细说明见：[docs/architecture/fulltext-acquisition.md](architecture/fulltext-acquisition.md)

### 4.2 PDF解析

- 从 `local_pdf_path` 读取本地 PDF
- 输出面向人阅读的 `Markdown`
- 输出面向程序处理的结构化 `JSON`
- 保留章节、段落、公式、表格、图注、参考文献等结构信息

### 4.3 结构化抽取

围绕电力系统研究，优先抽取以下高价值字段：

- 研究问题
- 主主题分类
- 方法分类
- 数学模型类型
- benchmark / 算例名称
- 节点规模与时间尺度
- 求解器与求解时间
- baseline 方法
- 作者声称改进点
- 证据摘录

### 4.4 单篇 Markdown 知识卡片

为每篇已解析论文生成研究笔记，至少包含：

- 基本信息
- 研究问题
- 方法与模型
- 算例规模
- 求解器与耗时
- 对比方法
- 主要贡献
- 局限性
- 原文证据

## 5. 中期目标

### 5.1 SOTA 对比知识库

围绕主题自动汇总：

- 方法对比表
- benchmark 对比表
- 求解时间对比表
- baseline 与 claim 对比表

### 5.2 研究脉络可视化

围绕“主题、方法、算例、指标、代表论文”构建研究脉络视图。

预期视图包括：

- 主题演化时间线
- 主题-方法热力图
- benchmark 规模梯度图
- 方法谱系图
- SOTA 对比面板

### 5.3 专题知识页

按主题生成专题综述页，例如：

- 稳定性
- 电力市场
- 综合能源
- 配电网与微电网
- 储能与新能源并网

## 6. 远期目标

- 支持本地模型辅助摘要、归纳与综述写作
- 支持问题驱动式检索与问答
- 支持对多篇论文做阶段性进展总结
- 支持围绕指定主题自动生成 related work 草稿
- 支持更细粒度的 benchmark 与 solver 横向比较

## 7. 核心设计原则

- 本地优先：文献索引、PDF、抽取结果和知识库均优先落地到本机
- 真实文献优先：AI 生成内容应尽量建立在已入库真实论文之上，而不是脱离证据自由发挥
- 防幻觉优先：系统核心价值之一是降低幻觉，提升结论的可核查性与可追溯性
- 证据可追溯：所有关键抽取字段尽量保留原文证据
- 结构化优先：优先形成稳定字段，再做大模型总结
- AI 可调用：知识组织方式不仅要适合人阅读，也要适合 AI 稳定检索、比对和复用
- 面向科研：重点服务综述、选题、对比、复现判断，不追求通用商业搜索体验
- 渐进增强：先保证“可用”，再逐步提升“完整”和“智能”

## 8. 暂不优先

- 自动化绕过授权边界下载受版权约束的全文
- 构建通用学术社交网络
- 做复杂前端框架重构
- 针对非电力系统领域做泛化优化
- 追求完全自动、无需人工复核的抽取结果

## 9. 建议的功能阶段

### Phase 1

`检索 -> 入库 -> 下载队列 -> PDF绑定`

### Phase 2

`PDF -> Markdown / JSON -> 单篇结构化抽取`

### Phase 3

`专题对比表 -> SOTA对比 -> Markdown知识库`

### Phase 4

`研究脉络可视化 -> 本地模型辅助综述`
